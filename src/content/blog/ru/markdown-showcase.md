Эта публикация представляет собой наглядный справочник по markdown для блога `Academic-Portfolio`. Если вас в первую очередь интересует отображение блоков кода, перейдите сразу к разделу «Блоки кода» ниже.

## Заголовки и строчное форматирование

Markdown поддерживает **жирный текст**, *курсивный текст*, ***комбинированное выделение***, ~~зачёркивание~~ и строчный код, например `npm run build`.

Стандартные строчные ссылки работают, как [Astro](https://astro.build/), а ссылки в референсном стиле также подходят для более длинных текстов, например [документация Astro][astro-docs]. Обычные URL-адреса, такие как https://docs.astro.build, также отображаются как ссылки в данной конфигурации.

## Блоки цитат

> Markdown лучше всего работает, когда исходный текст остаётся читаемым, а отрендеренная страница — удобной для просмотра.
>
> Это делает его особенно полезным для технических заметок, академического письма, списков изменений и руководств.

## Списки

### Маркированные и нумерованные списки

- Составьте черновик публикации в `src/content/blog`.
- Добавьте заголовки, чтобы оглавление содержало полезные якорные ссылки.
- Проверьте отрендеренный результат в браузере.

1. Напишите метаданные (frontmatter) статьи.
2. Добавьте образцы markdown-блоков.
3. Соберите сайт и убедитесь, что страница отображается корректно.

### Вложенные списки

- Markdown также поддерживает вложенные структуры.
  - Это полезно для создания планов.
  - Также помогает при документировании шагов с подпунктами.
- Цель здесь — просто сделать рендеринг удобным для проверки.

## Блоки кода

Следующие примеры позволяют легко сравнить различные стили огороженных блоков кода в текущей теме. Каждый блок отображает индикатор языка слева и кнопку копирования в правом верхнем углу.

```bash
npm install
npm run dev
npm run build
```

```ts
type Publication = {
  title: string;
  year: number;
  tags: string[];
};

const recentPublications = (items: Publication[]) =>
  items.filter((item) => item.year >= 2024);
```

```astro
---
const highlights = ['Markdown', 'Shiki code blocks', 'Generated TOC'];
---

<ul>
  {highlights.map((item) => <li>{item}</li>)}
</ul>
```


## HTML внутри Markdown

Встроенный HTML также доступен, когда вам нужно нечто, что нельзя чисто выразить средствами одного лишь markdown.

<details>
  <summary>Раскрываемый блок</summary>
  <p>Это полезно для раскрытий, разделов FAQ или необязательных заметок по реализации внутри публикации.</p>
</details>
